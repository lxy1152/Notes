时间复杂度

发表于2021-02-16|更新于2021-04-19|LeetCode概念梳理

概念

算法的时间复杂度是一个函数，它定性描述该算法的运行时间，常用大 OOO 符号来表示。如 O(n)O(n)O(n)、 O(n2)O(n^2)O(n2) 等。但通常时间复杂度的表达式中会含有常数项和低阶项，如： O(2n5+2n+1)O(2n^5+2n+1)O(2n5+2n+1)，对于这种情况通常是忽略系数、低阶项和常数项，只关注于高阶项，即在趋于无穷时的变化趋势。

如果算法与规模 n 呈对数阶变化，那么就可以用 O(log⁡n)O(\log n)O(logn) 来表示它的时间复杂度。由于通过 2×x=n2\times x=n2×x=n 可以推出 x=log⁡2nx=\log_2nx=log2​n，即执行 log⁡2n\log_2nlog2​n次后会退出循环。所以得到时间复杂度为：O(log⁡2n)O(\log_2n)O(log2​n)，即 O(log⁡n)O(\log n)O(logn)。

如果算法与规模 n 呈指数阶变化，那么就可以用 O(an)O(a^n)O(an) 来表示它的时间复杂度。比如下面的函数（计算斐波那契数列第 n 项的值）中，时间复杂度就是 O(2n)O(2^n)O(2n)。

可以参考快速排序算法，平均时间复杂度就是 O(nlog⁡n)O(n\log n)O(nlogn)。
